# Zava Learning — Corporate Reporting & Brand Standard

The house style for every incident deliverable (RCA report, executive deck, email, Teams
notification). Reporting skills retrieve this with `SearchMemory` (memory name `zava-brand`)
and apply it verbatim. Goal: every artifact looks like it came from one professional, calm,
executive-ready company.

## Voice & tone
- Clear, factual, calm. No blame, no speculation stated as fact, no internal jargon.
- Lead with impact and outcome; put mechanism and evidence after.
- Past tense for what happened; present/future tense for recommendations.
- Quantify everything you can (duration, error rate, users affected, % recovered).
- Audience-layer the language: executives read the summary; engineers read the appendix.

## Brand palette (hex)
| Token | Hex | Use |
|---|---|---|
| Zava Indigo (primary) | `#4B2E83` | Title bars, headings, primary accents |
| Zava Teal (secondary) | `#00A39A` | "After" / recovery series, links, callouts |
| Slate (neutral text) | `#2E3440` | Body text |
| Mist (neutral fill) | `#F4F4F8` | Table zebra, slide backgrounds |
| Success | `#2E8B57` | Healthy / resolved |
| Warning | `#E8A317` | Degraded / at-risk |
| Critical | `#C0392B` | Outage / "before" series |

Charts: "before" series in Critical red, "after" series in Zava Teal, baseline/target as a
dashed Slate line. Never use red/green as the only differentiator — also label series.

## Typography
- Headings: **Segoe UI Semibold**. Body: **Segoe UI** / system sans. Monospace for IDs,
  resource names, KQL, and code: `Cascadia Mono` / `Consolas`.

## Logo & header
- Wordmark: **ZAVA Learning**. Indigo wordmark on light, white wordmark on Indigo.
- Header line on every artifact: `ZAVA Learning · Site Reliability · <Report Type> · <UTC date>`.
- Footer: `Confidential — Internal Use Only · Generated by the Zava SRE Agent`.

## Severity labels (use consistently)
`SEV1 Critical` (Critical red) · `SEV2 Major` (Warning amber) · `SEV3 Minor` (Slate).
Severity is impact-based (users affected × functionality lost), never the alert name.

## Layout specs
- **Slide (16:9):** Indigo title bar with white title (28–32pt) + ZAVA wordmark top-right;
  one idea per slide; max ~6 bullets / ~7 words each; charts > tables > paragraphs.
  Canonical deck order is defined in `zava-report-template`.
- **Email (HTML):** Indigo header band with wordmark; single column, max ~640px; H2 section
  headings in Indigo; one summary table; "Recommended actions" as a checklist; brand footer.
- **Teams (Adaptive Card):** Title `🟣 ZAVA SRE — <symptom> <status>`; FactSet for
  Severity / Status / Duration / Users impacted; one-line root cause; up to 3 next steps;
  Action.OpenUrl buttons for PR, ServiceNow CR, and PagerDuty incident.

## Do / Don't
- DO render real visuals with the built-in Plot* tools (or `ExecutePythonCode` + matplotlib /
  python-pptx) — never describe a chart you could draw.
- DO show absolute numbers **and** deltas (e.g. `5.2% → 0.3% (−94%)`).
- DON'T reveal root cause in any title or alert reference — the title is symptom-only.
- DON'T include credentials, tokens, full connection strings, or PII.
- DON'T claim a fix is verified without the post-fix metric to prove it.
